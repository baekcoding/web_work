* git으로 관리할 대상 파일을 모든 파일을 대상으로 하면

안좋은 경우가 있다.(거의 대부분 그렇다.)

설정 파일은 git의 추적대상에서 제거할 필요가 있다.

여기서 설정이란

1. 이클립스 설정
2. java 설정(java version, 환경 등등)

- 모든 설정은 다 제외하고 코딩한 최소한의 파일만 add, commit 해서
git hub에 올려야 한다.
- 설정을 파일 추적대상에서 제외하는 이유는 git hub에 올린 프로젝트를
내려받은 개발 환경이 git hub에 push했었던 개발환경과 정확히 일치하지 않으면
에러가 발생하기 때문이다.
- git hub에 올라간 프로젝트는 모든 설정 파일이 빠져있기 때문에
java 프로젝트의 모양을 갖추고 있지 못해서 바로 import가 되지 않는다.
내려받은 곳에서 자기 환경에 맞게끔 설정을 만들면서 import 해야 한다.
(대부분 자동으로 됨)
- 그럼 도대체 어떤 파일을 제외해야 하는지 우리는 어떻게 알 수 있나?
구글에 gitignore 검색 > toptal.com 사이트 접속 > 검색창에 windows, java, eclipse 추가해서 생성 > 생성된거 복사해서 이클립스에 git모드로 해서 Working Tree
내에 gitignore 문서에 붙여넣기 > .classpath, .project 내용 추가한 다음
> git staging에서 add해서 커밋
> 깃허브에서 저장소 만들고 url 복사 > 이클립스 java_work 우클릭해서 'push branch
master' 클릭 > 사용자명, 보안토큰 번호 입력해서 push 완료


필드는 주로 메소드에서 필요한 값들을 담고 있다.(전역변수 생성이 안되므로
생성자의 변수를 메소드에서 쓰고 싶을때 필드에 담아서 사용한다.)
